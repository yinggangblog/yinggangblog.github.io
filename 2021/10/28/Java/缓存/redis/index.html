分布式锁

分布式锁原理与应用

分布式锁基本原理

理解：就先当1000个人去占一个厕所（setIfAbsent（key,value）），厕所只能有一个人占到这个坑，占到这个坑其他人就只能在外面等待，等待一段时间后可以再次来占坑(调用此方法)，业务执行后，释放锁(delete(key))，那么其他人就可以来占这个坑

setNx 原子性 没有就放数据

setEx 设置过期时间

分布式锁演进 - 阶段一 没有设置锁的过期时间

setEx 设置锁的过期时间，即使没有删除，会自动删除
